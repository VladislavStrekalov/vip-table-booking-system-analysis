# Функциональные требования

## FR-1. Просмотр VIP-столиков

Система должна предоставлять клиенту возможность просмотра списка доступных VIP-столиков.


## FR-2. Просмотр доступности столика

Система должна отображать свободные временные интервалы для выбранного VIP-столика.


## FR-3. Создание бронирования

Система должна позволять клиенту создавать бронирование VIP-столика на выбранную дату и время.


## FR-4. Проверка доступности

Перед созданием бронирования система должна проверять доступность выбранного столика на указанный временной интервал.


## FR-5. Предотвращение двойного бронирования

Система не должна позволять создавать два активных бронирования для одного столика на пересекающееся время.


## FR-6. Отмена бронирования

Система должна позволять клиенту отменять собственное бронирование.


## FR-7. Подтверждение бронирования

Система должна предоставлять менеджеру возможность подтверждать созданные бронирования.


## FR-8. Управление VIP-столиками

Менеджер должен иметь возможность создавать, изменять и удалять информацию о VIP-столиках.


## FR-9. Просмотр списка бронирований

Менеджер должен иметь возможность просматривать список всех бронирований.


## FR-10. Уведомление клиента

После создания или отмены бронирования система должна уведомлять клиента об изменении статуса бронирования.
