# Система  бронирования VIP-столиков в ресторане
## Описание проекта

Данный проект представляет собой анализ системы бронирования VIP-столиков в ресторане.

Система предназначена для того, чтобы клиенты могли бронировать VIP-столики на определённую дату и время, а сотрудники ресторана могли управлять бронированиями и контролировать загруженность столиков.

Проект выполняется в учебных целях для изучения системного анализа, разработки требований, моделирования бизнес-процессов и проектирования информационных систем.

## Цель системы

Автоматизировать процесс бронирования VIP-столиков и обеспечить удобное взаимодействие между клиентами и персоналом ресторана.

## Роли пользователей

### Клиент

Клиент может:

* просматривать доступные VIP-столики;
* выбирать дату и время бронирования;
* создавать бронирование;
* отменять бронирование;
* просматривать информацию о своих бронированиях.

### Менеджер ресторана

Менеджер может:

* просматривать все бронирования;
* подтверждать бронирования;
* отменять бронирования;
* управлять VIP-столиками;
* контролировать занятость столиков.

## Основные возможности системы

* просмотр списка VIP-столиков;
* поиск свободных столиков;
* создание бронирования;
* подтверждение бронирования менеджером;
* отмена бронирования;
* предотвращение двойного бронирования.

## Планируемые артефакты проекта

В рамках проекта будут разработаны:

* функциональные требования;
* пользовательские сценарии (Use Cases);
* BPMN-диаграмма процесса бронирования;
* UML-диаграммы;
* ER-диаграмма;
* спецификация API;
* модель данных системы.
